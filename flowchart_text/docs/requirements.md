OCR 矢量还原系统需求规范 (Requirements)

1. 目标描述

将复杂的学术/技术插图（位图格式）还原为 draw.io (mxGraph XML) 可编辑的矢量格式。重点在于文字内容的极致准确性、空间位置的像素级对齐、以及字号大小的视觉一致性，无需进行语义理解。

2. 输入要求

• 格式: PNG, JPG, BMP 或高分辨率扫描 PDF。  
• 分辨率: 建议 300 DPI 以上，以确保 Azure 高分辨率模式能识别 12px 以下的小字。

3. 核心功能指标 (KPI)

• 内容准确性: 针对标准文本识别率 >99%，针对 LaTeX 公式识别率 >95%。  
• 坐标精度: 文字块的重心/基线偏移需控制在原图对应坐标的 ±2 像素以内。  
• 字号鲁棒性: 同一文本框内文字严禁出现因笔画差异导致的“字号抖动”，整图字号分级需符合原图视觉档位。  
• 输出格式: 标准的.drawio XML (mxGraph)，支持直接导入 draw.io 网页端或桌面端。

4. 约束条件

• 去语义化: 系统不应因猜测文字含义（如标题、正文）而擅自更改坐标或字号。  
• 单位统一: 必须建立从原图像素 (px) 到矢量画布磅值 (pt) 的线性归一化映射逻辑。
